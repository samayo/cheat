Git Flow

  • git switch -c feat/<desc> # start a feature branch (don’t commit on main)
  • git add -A && git commit -m "wip: <message>" # save progress early/often
  • git push -u origin HEAD # back up the branch remotely
  • git stash push -m "wip: <msg>" # park messy changes without committing (optional)
  • git switch main && git pull # update main before merging
  • git merge --squash feat/<desc> && git commit -m "feat: <summary>" # squash-merge feature into main
  • git tag -a vX.Y.Z -m "release: <notes>" # mark a stable checkpoint (use semver)
  • git push origin main --tags # push code and tags
  • git revert --no-edit vX.Y.Z..HEAD # rollback everything after the tag if needed
  • git switch -c release/X.Y.Z vX.Y.Z # hotfix branch from the tag; tag vX.Y.Z+1 when done
